NETWORK · DISCOVERY · LAN · DDNS

The bridge now listens on 0.0.0.0 (all interfaces) by default. This means it accepts connections from your local network, not just localhost. Use GET /network/info to discover the correct URL to use from another machine.

LAN ACCESS: Your bridge's LAN IP is typically 192.168.x.x or 10.x.x.x. Use GET /network/info to get it automatically. From another computer on the same network, replace 127.0.0.1 with your LAN IP.
GET/network/infoDiscover all bridge addresses

Returns all ways to reach this bridge: local, LAN, and DDNS if configured. Call this first from any external client.

// Response 200
{
  ok:        true,
  version:   "3.11.0",
  local:     "http://127.0.0.1:3747",     // always works on this machine
  lan:       "http://192.168.1.42:3747",   // use this from other LAN devices
  lanIp:     "192.168.1.42",
  ddns:      "http://home.mydomain.com:3747" // null if not configured
  port:      3747,
  shortId:   "a3f2b1c8",               // 8-char bridge fingerprint
  endpoints: ["/health", "/pulse", ...],
  ts:        1234567890123
}

DDNS / Remote Configuration

To reach your bridge from the internet (or a different network), configure a DDNS hostname:

// Configure DDNS via POST /config
POST /config
{ "noipUrl": "http://home.mydomain.com" }

// Then /network/info will include:
{ "ddns": "http://home.mydomain.com:3747" }

BrainOS has a DDNS panel (NETWORK → DDNS) that lets you register a hostname via No-IP or any custom DDNS provider. Once set, BrainOS and other clients auto-discover the DDNS URL from /network/info.

AUTH · HANDSHAKE · TOKEN LIFECYCLE

Tokens are optional for localhost connections. Required for remote/multi-node deployments. Every system connecting to the bridge should do a handshake first — it registers your presence and returns a 60s token.

POST/guardian/handshakeRegister + receive token
// Request
{
  instanceId:   "myapp-abc123",          // your unique ID (format: {prefix}-{uuid})
  logicalId:    "myapp",                 // human label shown in mesh
  challenge:    "any-random-string",
  capabilities: ["read", "write"],      // optional
  version:      "1.0.0"                  // optional
}

// Response 200
{
  ok:          true,
  token:       "eyJ...",               // 60s TTL — include in X-Guardian-Token header
  version:     "3.11.0",
  instanceId:  "bridge-{uuid}",
  nodes:       [ /* known mesh nodes */ ]
}
POST/guardian/heartbeatRefresh token TTL
// Request — include X-Guardian-Token header
{ ts: Date.now() }

// Response 200
{ ok: true, ttl: 60000 }

PULSE · HEARTBEAT · PRESENCE

Every node in the mesh sends a pulse every 1–2 seconds. The bridge emits guardian-pulse on the SSE bus, which BrainOS canvas uses to keep nodes alive and colored. Nodes that stop pulsing go idle at 15s and dead (red) at 30s.

POST/pulseHeartbeat — send every 1.5s
// Request
{
  instanceId:  "guardian-6f5a028e-4589...",  // your full instanceId
  logicalId:   "guardian",                   // display name in mesh
  nexusUrl:    "http://127.0.0.1:3747",
  bridgeUrl:   "http://127.0.0.1:3747",
  status:      "online",
  ts:          1234567890123
}

// Response 200
{ ok: true, version: "3.11.0", ts: 1234567890123 }

// GET /pulse — probe only (no body required)
// Response 200
{ ok: true, version: "3.11.0", ts: 1234567890123 }
POST/ingestIR layer causal event ingest

Used by the Guardian IR layer to ingest causal events. Emits guardian.ir.ingest on the SSE bus. Any JSON payload accepted.

CALLTO · REGISTRATION · EXEC

A callto is a reference to an element on a web page captured by Guardian. It has a selector, URL, and action. When you call POST /userscript/callto/:id/exec, Guardian performs the action (click, type, submit) on the live page element.

POST/userscript/calltoRegister a callto
// Request
{
  id:       "callto-{uuid}",
  selector: "div.my-2.5.flex.min-h-14",  // CSS selector
  label:    "ChatGPT input",
  url:      "https://chatgpt.com",
  host:     "chatgpt.com",
  action:   "type",                     // click | type | submit | listen
  ts:       1234567890123
}

// Response 200
{ ok: true, calltoId: "callto-{uuid}" }
GET/userscript/calltoList all registered calltos
// Response 200
{
  calltos: [
    { id, selector, label, url, host, action, ts },
    ...
  ]
}
POST/userscript/callto/:id/execExecute callto action

Guardian performs the callto's action on the live page. Response arrives asynchronously via GET /events as guardian.link.event.

// Request
{
  event: {
    type:   "exec",
    text:   "Hello world",    // text to type (if action=type)
    source: "nexus"
  }
}

// Response 200
{ ok: true, calltoId: "callto-abc123" }

// Then listen on GET /events for:
{ type: "guardian.link.event", text: "AI response text here", calltoId: "..." }

  // d.ts      — unix timestamp
};
evs.onerror = () => { evs.close(); setTimeout(reconnect, 3000); };

PIPELINE · MULTI-STEP EXECUTION

Pipelines chain multiple AI steps. Each step's output becomes the next step's input. Guardian steps dispatch async; direct API steps run synchronously.

// Pipeline is executed via sequential POST /v1/messages calls
// Each step:
const step = {
  provider: "guardian",      // or ollama|deepseek|claude|chatgpt|auto
  agentId:  "chatgpt",       // required when provider=guardian
  intent:   "chat",
  model:    "deepseek-r1:8b", // optional
  system:   "You are...",     // optional system prompt
  optional: false,            // if true, skip on error instead of failing
  transform: "output.toUpperCase()"  // optional JS expression applied to output
};

// Execution flow:
// Step 1: POST /v1/messages {provider, messages:[{role:'user', content: initialInput}]}
// Step 2: POST /v1/messages {provider, messages:[{role:'user', content: step1Output}]}
// Step N: uses step N-1 output as input

// Guardian step response (202):
{ status: "dispatched", agentId: "chatgpt" }
// → listen on GET /events for guardian.link.event to get the actual response

Deriving a ShortId

const instanceId = 'guardian-6f5a028e-4589-43f3-9a61-0c83ce2e9715';
// Method 1: strip prefix, take first 8 chars of UUID
const shortId = instanceId.replace(/^[a-z]+-/, '').slice(0, 8);
// → "6f5a028e"

// Method 2: stable fingerprint from LAN IP + port (for bridge nodes)
const shortId = require('crypto')
  .createHash('sha256').update(lanIp + port).digest('hex').slice(0, 8);

Share to Connect

The shortId is what you share to connect a node. Right-click any node in BrainOS canvas → copy shortId → send to another system. They can then reference your node by shortId in their connection config.

INTEGRATION EXAMPLES

Minimum Viable Client (copy-paste ready)

// Works in browser, Node.js, BrainOS, Electron, Causal-Nexus adapters
const BRIDGE = 'http://192.168.1.42:3747'; // replace with your LAN IP from /network/info
const CLIENT_ID = 'myapp-' + crypto.randomUUID();

// 1. Discover bridge address
async function discoverBridge() {
  // Try local first, then probe common LAN ranges
  const urls = [
    'http://127.0.0.1:3747',
    'http://192.168.1.42:3747', // update this
  ];
  for (const url of urls) {
    try {
      const r = await fetch(url + '/network/info', { signal: AbortSignal.timeout(1500) });
      if (r.ok) { const d = await r.json(); return d; }
    } catch (_) {}
  }
  return null;
}

// 2. Handshake
async function handshake(bridgeUrl) {
  const r = await fetch(bridgeUrl + '/guardian/handshake', {
    method: 'POST', headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ instanceId: CLIENT_ID, logicalId: 'myapp', version: '1.0.0' })
  });
  return (await r.json()).token;
}

// 3. Pulse loop
function startPulse(bridgeUrl, token) {
  return setInterval(() => fetch(bridgeUrl + '/pulse', {
    method: 'POST', headers: { 'Content-Type': 'application/json', 'X-Guardian-Token': token },
    body: JSON.stringify({ instanceId: CLIENT_ID, logicalId: 'myapp', status: 'online', ts: Date.now() })
  }).catch(() => {}), 1500);
}

// 4. Subscribe to events
function subscribeEvents(bridgeUrl, onEvent) {
  const evs = new EventSource(bridgeUrl + '/events');
  evs.onmessage = e => { try { onEvent(JSON.parse(e.data)); } catch (_) {} };
  evs.onerror   = () => { evs.close(); setTimeout(() => subscribeEvents(bridgeUrl, onEvent), 3000); };
  return evs;
}

// 5. Send via Guardian (no API key needed)
async function askGuardian(bridgeUrl, agentId, text) {
  const r = await fetch(bridgeUrl + '/v1/messages', {
    method: 'POST', headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      provider: 'guardian', agentId,
      messages: [{ role: 'user', content: text }], intent: 'chat'
    })
  });
  if (r.status === 202) return 'dispatched'; // response via SSE
  const d = await r.json(); return d.content?.[0]?.text || d;
}

// Bootstrap
(async () => {
  const info = await discoverBridge();
  const bridgeUrl = info?.lan || 'http://127.0.0.1:3747';
  const token = await handshake(bridgeUrl);
  startPulse(bridgeUrl, token);
  subscribeEvents(bridgeUrl, d => {
    if (d.type === 'guardian.link.event') console.log('AI:', d.text);
    if (d.type === 'guardian.callto.captured') console.log('New callto:', d.selector);
  });
  await askGuardian(bridgeUrl, 'chatgpt', 'Hello from my app!');
})();

Causal-Nexus Adapter

// Wire bridge SSE events into a URCK kernel
import { createKernel } from 'causal-nexus';
const kernel = createKernel();
const BRIDGE = 'http://192.168.1.42:3747';

const evs = new EventSource(BRIDGE + '/events');
evs.onmessage = e => {
  const d = JSON.parse(e.data);
  // Map bridge events → URCK kernel ingestion
  if (d.type?.startsWith('guardian')) {
    kernel.ingest(d.type.replace(/\./g, ':'), d.payload || d, 'bridge');
  }
};
